# Resumo da Aula – Paradigma ReAct (Reasoning + Acting)

## O que é o paradigma ReAct?
- **ReAct** = **Reasoning** (raciocinar) + **Acting** (atuar)  
- Em português ficaria algo como **Ra-a-ti** (Raciocinar e Atuar)  
- É um paradigma relativamente novo (criado há cerca de 1–2 anos) que revolucionou o uso de LLMs e possibilitou o “boom” dos agentes de IA.

## Por que o ReAct é tão importante?
Antes do ReAct (ex.: primeiras versões do ChatGPT):
- A IA recebia o prompt e respondia **direto**, sem “pensar”  
- Resultado: respostas rápidas, mas muitas vezes erradas, incoerentes ou inventadas (hallucinations fortes)

Com o ReAct:
- A IA é forçada a **raciocinar primeiro** antes de dar a resposta final  
- Depois do raciocínio, ela pode **executar ações** necessárias  
- Isso torna as respostas muito mais precisas e permite que a IA resolva problemas complexos de forma autônoma

## Como funciona o “Reasoning” (raciocínio)?
- A IA não raciocina como humano (é só uma forma de falar)  
- Na prática: em vez de prever a resposta diretamente a partir do prompt, a IA primeiro gera um **texto de raciocínio/pensamento**  
- Depois, usa esse raciocínio + o prompt original para gerar a resposta final  

Exemplo simples (soma 32 + 64):
- Sem raciocínio → pode dar uma resposta errada na lata (ex.: 50)  
- Com raciocínio → a IA gera passos intermediários (“somar unidades: 2+4=6”, “somar dezenas: 3+6=9”) → resposta correta (96)

É como se fossem **duas chamadas ao LLM**:
1. LLM gera o raciocínio a partir do prompt  
2. LLM gera a resposta final a partir do prompt + raciocínio  

O professor comentou que o **ChatGPT-4** deu um salto enorme em relação ao 3.5 exatamente por causa do melhor raciocínio.

## Como funciona o “Acting” (atuar)?
- Depois de raciocinar, a IA decide e executa **ações concretas** (chamadas de ferramentas/tools)  
- Essas ações podem ser: consultar APIs, fazer buscas, acessar dados, etc.

### Exemplo prático: “Como está o tempo hoje?”
Passos que a IA faz com ReAct:
1. **Reasoning** (raciocínio interno):  
   - O usuário quer a previsão do tempo atual  
   - Preciso saber a localização do usuário  
   - Preciso saber a data de hoje  
   - Preciso consultar uma API de clima  
   - Depois disso, posso formatar a resposta

2. **Acting** (ações reais):  
   - Obter localização (perguntar ao usuário ou usar IP)  
   - Obter data atual (função interna getDate)  
   - Fazer requisição à API de clima (ex.: ClimaTempo)  
   - Gerar resposta final com base nos dados reais

Resultado: antes (sem ReAct) a IA inventava uma previsão ou dizia que não sabia. Agora ela responde corretamente.

## Conclusão
- O ciclo **ReAct** (raciocinar → atuar → raciocinar de novo se necessário) é o que permite que LLMs se tornem verdadeiros **agentes autônomos**  
- Todas as IAs modernas (ChatGPT, Gemini, Claude etc.) já operam com esse paradigma por padrão  
- Próximo passo da aula: ver na prática como esse ciclo aparece nas respostas das IAs atuais e começar a entender o uso de ferramentas (tool use), que é o coração do “Acting”.
